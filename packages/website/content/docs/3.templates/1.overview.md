# Template Overview

Mail Magic uses Nunjucks syntax inside templates (`.njk`) for variables, logic, `extends`, and `include`. A
preprocessing step flattens template hierarchies and rewrites `asset('...')` references before storage.

## Directory Layout

Templates and assets are loaded from the external config tree rooted at `CONFIG_PATH`.

For each sending domain you typically have:

```text
${CONFIG_PATH}/
└── example.com/
    ├── assets/
    │   ├── images/
    │   │   └── logo.png
    │   └── files/
    │       └── banner.png
    ├── tx-template/
    │   ├── base.njk
    │   ├── welcome.njk
    │   └── partials/
    │       └── header.njk
    └── form-template/
        ├── base.njk
        ├── contact.njk
        └── partials/
            └── fields.njk
```

Notes:

- Transactional templates live under `${domain}/tx-template/`.
- Form templates live under `${domain}/form-template/`.
- Shared static files referenced with `asset('...')` live under `${domain}/assets/`.

## Locales

Locales affect:

- Which template record is selected by APIs (transactional send, asset uploads).
- Where Mail Magic expects the `.njk` file to live on disk when importing from `CONFIG_PATH`.

Locales do **not** affect public asset URLs (assets are served at `/asset/${domain}/...` with no locale segment).

### Filesystem layout for locale variants

If you have multiple locale versions of the same template, place them under a locale subdirectory:

```text
${CONFIG_PATH}/example.com/tx-template/en/welcome.njk
${CONFIG_PATH}/example.com/tx-template/fr/welcome.njk

${CONFIG_PATH}/example.com/form-template/en/contact.njk
${CONFIG_PATH}/example.com/form-template/fr/contact.njk
```

Includes and extends are resolved relative to the current template root. For example, inside
`tx-template/fr/welcome.njk`, this loads `tx-template/fr/base.njk`:

```njk
{% extends "base.njk" %}
```

### Locale and generated template paths

When template records are created without an explicit `filename`, Mail Magic generates a path that includes the locale
(when non-empty):

- Transactional template: `${domainSlug}/tx-template/${localeSlug}/${name}.njk`
- Form template: `${domainSlug}/form-template/${localeSlug}/${idname}.njk`

If you set `filename` yourself (e.g. in `init-data.json`), Mail Magic will not automatically inject the locale
directory. Include the locale in the `filename` value you provide if you want locale-specific paths.

### Transactional API: selecting the right locale

When you send a transactional message (`/v1/tx/message`), you can pass `locale`. Lookup order is:

1. Match `name + domain + locale` (the request locale)
2. Match `name + domain + deflocale` (domain default locale)
3. Match `name + domain + empty locale` (fallback to the locale-less record)

Practical implications:

- If you only have one locale version, store it with an empty locale and it acts as the universal fallback.
- If you have multiple locale versions, always pass `locale` to avoid the fallback selecting the wrong one.
- Setting `locale` on the domain record establishes a default for all API calls that omit `locale`.

Example:

```json
{
  "name": "welcome",
  "domain": "example.com",
  "locale": "fr",
  "rcpt": "user@example.com",
  "vars": { "name": "Jean" }
}
```

### Form API: locale vs form_key

Forms are ultimately selected by `form_key` on public submission:

- `POST /v1/form/template` stores the form template record under `(domain, idname, locale)` and returns a `form_key`.
- If you create `contact` in `en` and `fr`, you get two different `form_key` values.
- Public form submissions use `_mm_form_key` to pick the form; `_mm_locale` is exposed to the template as a value but
  does not override which form template is selected.

## How Hierarchy Works (Parent Templates + Blocks)

Use standard Nunjucks inheritance:

`tx-template/base.njk`:

```njk
<!doctype html>
<html>
  <head>
    <title>{{ title }}</title>
  </head>
  <body>
    {% block body %}{% endblock %}
  </body>
</html>
```

`tx-template/welcome.njk`:

```njk
{% extends "base.njk" %}

{% block body %}
  {% include "partials/header.njk" %}

  <p>Hello {{ name }}</p>

  <!-- Inline (CID) asset -->
  <img src="asset('images/logo.png', true)" alt="logo" />

  <!-- Linked (public URL) asset -->
  <img src="asset('files/banner.png')" alt="banner" />
{% endblock %}
```

## Includes

Includes use standard Nunjucks syntax:

```njk
{% include "partials/footer.njk" %}
```

Recommendations:

- Keep includes simple and "pure" (no side effects), and pass data via variables in the render context.
- Prefer a `partials/` folder and consistent naming across `tx-template/` and `form-template/`.
