# Template Variables

## Transactional Templates

The `/v1/tx/message` endpoint renders the stored template with a context containing:

| Variable | Description |
|---|---|
| `{{ name }}` (and other vars) | Your provided `vars` merged at the top level |
| `_rcpt_email_` | Current recipient email address |
| `_attachments_` | Map of uploaded field name to original file name (for runtime uploads) |
| `_vars_` | The original `vars` object |
| `_meta_` | Request metadata: `client_ip`, `received_at`, `ip_chain` |

### Example

```njk
<p>Hi {{ name }} ({{ _rcpt_email_ }})</p>
<p>Uploaded: {{ _attachments_.file1 }}</p>
<p>IP: {{ _meta_.client_ip }}</p>
```

You can also access `vars` through the top-level namespace directly:

```njk
<p>Hi {{ _vars_.first_name or _rcpt_email_ }},</p>
<a href="{{ _vars_.cta_url or '#' }}">Confirm email</a>
```

## Form Templates

Form templates are rendered with a different context, reflecting the public submission data:

| Variable | Description |
|---|---|
| `_fields_` | All non-system submitted fields (optionally filtered by `allowed_fields`) |
| `_mm_form_key` | The form key used for the submission |
| `_mm_locale` | Locale value from the public submission |
| `_mm_recipients` | Recipients requested by the public submission |
| `_rcpt_email_` | Primary recipient email address |
| `_rcpt_name_` | Primary recipient display name |
| `_rcpt_idname_` | Primary recipient idname |
| `_rcpt_idnames_` | Array of all resolved recipient idnames |
| `_attachments_` | Map of uploaded field name to original file name |
| `_files_` | Uploaded file objects |
| `_vars_` | Always `{}` (reserved for future use) |
| `_meta_` | Request metadata: `client_ip`, `received_at`, `ip_chain` |

### Example

```njk
<p>Name: {{ _fields_.name }}</p>
<p>Email: {{ _fields_.email }}</p>
<p>Form key: {{ _mm_form_key }}</p>
<p>Received from: {{ _meta_.client_ip | default('unknown') }}</p>
```

### Iterating over all submitted fields

```njk
{% if _fields_ %}
  <table>
    {% for field, value in _fields_ %}
    <tr>
      <td>{{ field | replace('_', ' ') | title }}</td>
      <td>{{ value }}</td>
    </tr>
    {% endfor %}
  </table>
{% endif %}
```

## The `allowed_fields` Filter

When `allowed_fields` is configured on a form template (via the authenticated API), `_fields_` is filtered to that
allowlist. These fields are always allowed even if not listed: `email`, `name`, `first_name`, `last_name` (so Reply-To
extraction still works).
