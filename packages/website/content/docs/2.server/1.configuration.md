# Configuration

Mail Magic is configured via environment variables plus the `CONFIG_PATH` directory on disk.

The full set of variables is documented in the package's `.env-dist` file:

```bash
node_modules/@technomoron/mail-magic/examples/.env-dist
```

## Core Variables

| Variable | Default | Description |
|---|---|---|
| `API_HOST` | `127.0.0.1` | Host address the server binds to |
| `API_PORT` | `3776` | Port the server listens on |
| `API_URL` | `http://127.0.0.1:3776` | Public base URL (used for asset URL generation) |
| `API_TOKEN_PEPPER` | — | **Required.** Secret used to HMAC API tokens before DB storage. Keep stable. |
| `CONFIG_PATH` | `./data` | Path to the directory containing `init-data.json` and domain folders |

## Database

| Variable | Default | Description |
|---|---|---|
| `DB_TYPE` | `sqlite` | Database type (`sqlite`) |
| `DB_NAME` | `./maildata.db` | SQLite database file path |
| `DB_FORCE_SYNC` | `true` | Force-sync the schema on startup |
| `DB_SYNC_ALTER` | `false` | Use `ALTER TABLE` for schema migrations instead of recreating |
| `DB_AUTO_RELOAD` | `false` | Re-import `init-data.json` and template files when they change on disk |
| `DB_LOG` | `false` | Log SQL queries |

## SMTP

| Variable | Default | Description |
|---|---|---|
| `SMTP_HOST` | `127.0.0.1` | SMTP server hostname |
| `SMTP_PORT` | `1025` | SMTP server port |
| `SMTP_SECURE` | `false` | Use TLS from the start (port 465 style) |
| `SMTP_REQUIRE_TLS` | `true` | Require STARTTLS. Set to `false` for local dev servers like MailHog |
| `SMTP_TLS_REJECT` | `true` | Reject self-signed TLS certificates. Set to `false` for local dev |
| `SMTP_USER` | — | SMTP auth username (optional) |
| `SMTP_PASS` | — | SMTP auth password (optional) |

## Templates and Assets

| Variable | Default | Description |
|---|---|---|
| `AUTOESCAPE_HTML` | `true` | Autoescape HTML in Nunjucks template rendering |
| `ASSET_PUBLIC_BASE` | — | Override the base URL used for public asset links (defaults to `API_URL`) |

## Uploads

| Variable | Default | Description |
|---|---|---|
| `UPLOAD_PATH` | `./{domain}/uploads` | Directory for uploaded file staging. `{domain}` is replaced at runtime |
| `UPLOAD_MAX` | — | Max size in bytes per uploaded file |

## Public Form Anti-Abuse

| Variable | Default | Description |
|---|---|---|
| `FORM_RATE_LIMIT_WINDOW_SEC` | — | Rate limit window size in seconds |
| `FORM_RATE_LIMIT_MAX` | — | Max requests per client IP per window. `0` disables rate limiting |
| `FORM_MAX_ATTACHMENTS` | — | Max number of files per form submission. `-1` = unlimited, `0` = no attachments |
| `FORM_KEEP_UPLOADS` | — | When `false`, uploaded files are deleted after processing |
| `FORM_CAPTCHA_PROVIDER` | — | CAPTCHA provider: `turnstile`, `hcaptcha`, or `recaptcha` |
| `FORM_CAPTCHA_SECRET` | — | Provider secret key. Enables CAPTCHA verification when set |
| `FORM_CAPTCHA_REQUIRED` | — | When `true`, require CAPTCHA tokens for all form submissions |

## Swagger / OpenAPI

| Variable | Default | Description |
|---|---|---|
| `SWAGGER_ENABLED` | `false` | Serve the OpenAPI spec at `/api/swagger` |

## Miscellaneous

| Variable | Default | Description |
|---|---|---|
| `NODE_ENV` | `development` | Node environment |
| `DEBUG` | `false` | Enable debug logging |

## Notes

- `API_TOKEN_PEPPER` is used to derive the stored token hash as `HMAC-SHA256(token, API_TOKEN_PEPPER)`. If you change
  this value, all existing tokens become invalid.
- `DB_AUTO_RELOAD=true` is convenient for development: changes to `init-data.json` or `.njk` template files are
  re-imported without a server restart.
- For SMTP with a local relay (e.g. MailHog, Mailpit), set `SMTP_REQUIRE_TLS=false` and `SMTP_TLS_REJECT=false`.
