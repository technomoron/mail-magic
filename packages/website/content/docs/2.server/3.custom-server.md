# Custom Server

The `mail-magic` binary is a thin wrapper around a public Node.js API. You can import that API directly to embed Mail
Magic in your own application, add custom routes, or load configuration without a `.env` file.

Mail Magic runs on **[Fastify](https://fastify.dev)**. The bootstrap object gives you direct access to the
`FastifyInstance` via `server.fastify`, so you can register Fastify plugins, hooks, and routes alongside the built-in
Mail Magic API.

## Core Functions

```ts
import {
  createMailMagicServer,  // build the server, do not start it yet
  startMailMagicServer,   // build + start in one call
} from '@technomoron/mail-magic'
```

Both accept two optional arguments:

```ts
createMailMagicServer(
  serverOverrides?,   // Partial<ApiServerConf> — override host, port, upload limits, …
  envOverrides?,      // Partial<MailStoreVars>  — override any env variable programmatically
)
```

Both return a `MailMagicServerBootstrap`:

```ts
type MailMagicServerBootstrap = {
  server: mailApiServer   // extends ApiServer — exposes server.fastify (FastifyInstance)
  store:  mailStore       // database + mail transporter handle
  vars:   MailStoreVars   // resolved configuration
}
```

## Example: Programmatic Startup

Replace `.env`-based configuration with values from your own config source.

```ts
import { startMailMagicServer } from '@technomoron/mail-magic'

const { vars } = await startMailMagicServer(
  // server overrides (optional)
  {
    apiHost: '0.0.0.0',
    apiPort: 3776,
  },
  // env overrides — any MailStoreVars key accepted
  {
    CONFIG_PATH:       './mail-data',
    API_TOKEN_PEPPER:  process.env.MAIL_MAGIC_PEPPER!,
    SMTP_HOST:         process.env.SMTP_HOST!,
    SMTP_PORT:         587,
    SMTP_SECURE:       false,
    SMTP_REQUIRE_TLS:  true,
    SMTP_USER:         process.env.SMTP_USER!,
    SMTP_PASSWORD:     process.env.SMTP_PASSWORD!,
    DB_TYPE:           'postgres',
    DB_HOST:           process.env.DB_HOST!,
    DB_NAME:           'mailmagic',
    DB_USER:           process.env.DB_USER!,
    DB_PASS:           process.env.DB_PASS!,
  }
)

console.log(`Mail Magic listening on ${vars.API_HOST}:${vars.API_PORT}`)
```

## Example: Adding Custom API Routes

Use `createMailMagicServer` to get the server instance before it starts, register additional API modules, then call
`server.start()` yourself.

A custom module extends `ApiModule` from `@technomoron/api-server-base` and returns routes from `defineRoutes()`.

```ts
import { createMailMagicServer } from '@technomoron/mail-magic'
import { ApiModule } from '@technomoron/api-server-base'
import type { ApiRoute } from '@technomoron/api-server-base'

class StatusModule extends ApiModule {
  namespace = 'status'

  defineRoutes(): ApiRoute[] {
    return [
      {
        method:  'get',
        path:    '/ready',
        auth:    { type: 'none', req: 'any' },
        handler: async () => [200, { ready: true, ts: Date.now() }],
      },
      {
        method:  'get',
        path:    '/version',
        auth:    { type: 'apikey', req: 'any' },
        handler: async () => [200, { version: process.env.npm_package_version }],
      },
    ]
  }
}

const { server } = await createMailMagicServer()
server.api(new StatusModule())
await server.start()
// GET /api/v1/status/ready  — unauthenticated health probe
// GET /api/v1/status/version — requires API key
```

Route `path` is relative to `/api/v1/<namespace>`. Authentication types:

| `type`     | Behaviour |
|------------|-----------|
| `'none'`   | No authentication required |
| `'maybe'`  | Auth parsed if present, but not required |
| `'apikey'` | Requires a valid API key header |
| `'yes'`    | Requires any valid auth |
| `'strict'` | Requires auth; rejects expired/insufficient tokens |

## Example: Fastify Plugins and Hooks

Access `server.fastify` directly to register any Fastify plugin or lifecycle hook before the server starts.

```ts
import { createMailMagicServer } from '@technomoron/mail-magic'
import fastifyRateLimit from '@fastify/rate-limit'
import fastifyCors from '@fastify/cors'

const { server } = await createMailMagicServer()

// Native Fastify plugins — register before server.start()
await server.fastify.register(fastifyCors, {
  origin: ['https://app.example.com'],
})

await server.fastify.register(fastifyRateLimit, {
  max:        100,
  timeWindow: '1 minute',
})

// Fastify lifecycle hook
server.fastify.addHook('onRequest', async (request) => {
  request.log.info({ url: request.url }, 'incoming request')
})

await server.start()
```

## Example: Express-style Middleware (compat)

`server.useExpress()` is a compatibility adapter that lets you mount Express-style middleware on the Fastify server.
Use it when you need a library that ships an Express middleware interface.

```ts
import { createMailMagicServer } from '@technomoron/mail-magic'
import morgan from 'morgan'

const { server } = await createMailMagicServer()

// Express-style middleware runs via the compat adapter
server.useExpress(morgan('combined'))

// Scoped to a path prefix
server.useExpress('/api/v1/form', (req, _res, next) => {
  console.log('form submission from', req.ip)
  next()
})

await server.start()
```

Prefer native Fastify plugins over the compat adapter where possible — they integrate more cleanly with Fastify's
request lifecycle and error handling.

## Example: Graceful Shutdown

```ts
import { createMailMagicServer } from '@technomoron/mail-magic'

const { server, store } = await createMailMagicServer()
await server.start()

async function shutdown(signal: string): Promise<void> {
  console.log(`${signal} received — shutting down`)
  // Fastify drain and close
  await server.fastify.close()
  // Close the database connection
  await store.api_db?.close()
  process.exit(0)
}

process.on('SIGTERM', () => void shutdown('SIGTERM'))
process.on('SIGINT',  () => void shutdown('SIGINT'))
```
