# Quick Start

## Step 1: Create a `.env`

```ini
API_TOKEN_PEPPER=change-me-please-use-a-long-random-string

CONFIG_PATH=./data

API_HOST=127.0.0.1
API_PORT=3776
API_URL=http://127.0.0.1:3776

SMTP_HOST=127.0.0.1
SMTP_PORT=1025
SMTP_SECURE=false
SMTP_REQUIRE_TLS=false
SMTP_TLS_REJECT=false
```

You can copy the full example env from the package:

```bash
cp node_modules/@technomoron/mail-magic/examples/.env-dist .env
```

## Step 2: Create a config directory

Copy the example data directory as a starting point:

```bash
cp -r node_modules/@technomoron/mail-magic/examples/data ./data
```

This gives you a working `init-data.json` and `example.test` domain with transactional and form templates already
set up.

The directory structure looks like this:

```text
data/
  init-data.json
  example.test/
    assets/
      images/logo.png
    tx-template/
      en/
        welcome.njk
    form-template/
      en/
        contact.njk
```

## Step 3: Start the server

```bash
mail-magic --env ./.env
```

## First API Calls

### Ping

```bash
curl http://localhost:3776/api/v1/ping
```

### Send a transactional email

The example config ships with a token `example-token` and domain `example.test`:

```bash
curl -X POST http://localhost:3776/api/v1/tx/message \
  -H "Authorization: Bearer apikey-example-token" \
  -H "Content-Type: application/json" \
  -d '{
    "domain": "example.test",
    "name": "welcome",
    "locale": "en",
    "rcpt": "user@example.test",
    "vars": { "first_name": "Ada" }
  }'
```

### Submit a form

Get the `form_key` for the example contact form from your `data/init-data.json`, then:

```bash
curl -X POST http://localhost:3776/api/v1/form/message \
  -H "Content-Type: application/json" \
  -d '{
    "_mm_form_key": "<form_key from init-data.json>",
    "name": "Kai",
    "email": "kai@example.test",
    "message": "Hello from the contact form"
  }'
```
