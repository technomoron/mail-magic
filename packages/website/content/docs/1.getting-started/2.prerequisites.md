# Prerequisites

Mail Magic has three runtime dependencies: a supported Node.js version, a SQL database, and an outgoing SMTP server.

## Node.js

Mail Magic requires **Node.js 20 or later**. Node 22 LTS is recommended for production.

Verify your version:

```bash
node --version
```

If you need to manage multiple Node versions, [nvm](https://github.com/nvm-sh/nvm) or [fnm](https://github.com/Schniz/fnm) work well.

## SQL Database

Mail Magic uses Sequelize and supports three database backends:

| Backend | `DB_TYPE` value | Notes |
|---------|----------------|-------|
| SQLite | `sqlite` | Default. Zero-config, single file — fine for small deployments. |
| MySQL / MariaDB | `mysql` | Good choice for teams already running MySQL or MariaDB. |
| PostgreSQL | `postgres` | Good choice for teams already running Postgres. |

For **SQLite** nothing extra is needed — the database file is created automatically on first start.

For **MySQL / MariaDB** or **PostgreSQL**, create an empty database and a user with full privileges on it before
running Mail Magic:

::code-group
```sql [MySQL / MariaDB]
CREATE DATABASE mailmagic CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'mailmagic'@'localhost' IDENTIFIED BY 'your-password';
GRANT ALL PRIVILEGES ON mailmagic.* TO 'mailmagic'@'localhost';
FLUSH PRIVILEGES;
```

```sql [PostgreSQL]
CREATE DATABASE mailmagic;
CREATE USER mailmagic WITH PASSWORD 'your-password';
GRANT ALL PRIVILEGES ON DATABASE mailmagic TO mailmagic;
```
::

Then set the corresponding environment variables:

```ini
DB_TYPE=mysql       # or postgres
DB_HOST=127.0.0.1
DB_NAME=mailmagic
DB_USER=mailmagic
DB_PASS=your-password
```

Mail Magic creates and migrates its own tables on startup — you do not need to run schema migrations manually.

## Outgoing SMTP Server

Mail Magic delivers email through any standard SMTP server. You need:

- **Host** and **port** — provided by your email service or mail server
- **Credentials** — username and password (if your server requires authentication)
- **TLS mode** — whether the connection uses implicit TLS (port 465), STARTTLS (port 587), or plain (port 25)

Common choices:

| Option | Good for |
|--------|----------|
| [Postfix](https://www.postfix.org/) / [Exim](https://www.exim.org/) | Self-hosted, full control |
| [Mailpit](https://mailpit.axllent.org/) | Local development (catches all mail) |
| [Resend](https://resend.com), [Postmark](https://postmarkapp.com), [SendGrid](https://sendgrid.com) | Managed deliverability |
| AWS SES, Google Workspace SMTP relay | Cloud-hosted |

The relevant environment variables:

```ini
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_SECURE=false        # true for implicit TLS (port 465)
SMTP_REQUIRE_TLS=true    # require STARTTLS upgrade
SMTP_TLS_REJECT=true     # set false only for self-signed certs in dev
SMTP_USER=your-smtp-user
SMTP_PASSWORD=your-smtp-password
```

For local development with Mailpit:

```ini
SMTP_HOST=127.0.0.1
SMTP_PORT=1025
SMTP_SECURE=false
SMTP_REQUIRE_TLS=false
SMTP_TLS_REJECT=false
```

Once you have these three dependencies in place, move on to [Installation](/docs/getting-started/installation).
