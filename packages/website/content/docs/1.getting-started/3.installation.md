# Installation

## Install

```bash
npm install @technomoron/mail-magic
```

The package ships a `mail-magic` CLI bin that loads a `.env` file and starts the server. It also ships a runnable
example environment and config tree under `examples/`.

## Setting Up the Environment

Create a `.env` file. If you installed from npm, start from the example:

```bash
cp node_modules/@technomoron/mail-magic/examples/.env-dist .env
```

The example env is tuned for local development. For internet-facing form deployments, review the
[form security guide](/docs/forms/security) and set stricter values for rate limiting, CAPTCHA, attachment limits, and
SMTP TLS verification.

### Required environment variables

At a minimum you must set:

```ini
# REQUIRED. Keep stable: used to HMAC API tokens before DB lookup.
API_TOKEN_PEPPER=change-me-please-use-a-long-random-string

CONFIG_PATH=./data

API_HOST=127.0.0.1
API_PORT=3776
API_URL=http://127.0.0.1:3776

SMTP_HOST=127.0.0.1
SMTP_PORT=1025
SMTP_SECURE=false
SMTP_REQUIRE_TLS=false
SMTP_TLS_REJECT=false
```

### Full environment variable reference

See the [Server Configuration](/docs/server/configuration) page for the complete list of environment variables.

## Config Directory Structure

`CONFIG_PATH` points at a directory containing `init-data.json` plus per-domain subfolders:

```text
data/
  init-data.json
  example.com/
    assets/
      images/
        logo.png
    tx-template/
      welcome.njk
      partials/
        header.njk
    form-template/
      contact.njk
      partials/
        fields.njk
```

To use the ready-made example config as a starting point:

```bash
cp -r node_modules/@technomoron/mail-magic/examples/data ./data
```

Or from the monorepo:

```bash
cp -r packages/server/examples/data ./data
```

Assets referenced via `asset('...')` in templates must live under `<CONFIG_PATH>/<domain>/assets/...`.

## Starting the Server

```bash
mail-magic --env ./.env
```

The server starts on the configured `API_HOST:API_PORT` (default `127.0.0.1:3776`).

Verify it is running:

```bash
curl http://localhost:3776/api/v1/ping
```
